### MRO (Method Resolution Order)

El **MRO (Method Resolution Order)** en Python es el orden en el que el int茅rprete de Python busca m茅todos y atributos en una jerarqu铆a de clases, especialmente en casos de herencia m煤ltiple. El MRO determina cu谩l m茅todo o atributo se ejecuta o se accede cuando hay ambig眉edades debido a que una clase hereda de varias clases base.

#### 驴Por qu茅 es importante el MRO?

En Python, una clase puede heredar de m煤ltiples clases (herencia m煤ltiple). Esto puede llevar a situaciones en las que dos o m谩s clases base define un m茅todo o atributo con el mismo nombre. El MRO resuelve este problema definiendo un orden claro y predecible para buscar m茅todos y atributos.

#### 驴C贸mo funciona el MRO?

Python utiliza un algoritmo llamado **C3 Linearizaci贸n** para calcular el MRO. Este algoritmo garantiza que:

1. El orden sea consistente y predecible.
2. Se respecte el orden de herencia definido por el programador.
3. No haya ciclos en la jerarqu铆a de herencia.

El MRO sigue estos principios:

- Una clase siempre tiene prioridad sobre sus clases base.
- Si una clase hereda de m煤ltiples clases, el orden en que se espec铆fican las clases base importa.
- Se respecta el orden de herencia de izquierda a derecha.

**Ejemplo b谩sico de MRO:**

````python

    def metodo(self):
        print("M茅todo de A")

class B(A):
    def metodo(self):
        print("M茅todo de B")

class C(A):
    def metodo(self):
        print("M茅todo de C")

class D(B, C):
    pass

objeto = D()
objeto.metodo()  # 驴Qu茅 m茅todo se ejecuta?
````

En este caso, el MRO de la clase `D` se calcula como:

1. `D` (la clase actual).
2. `B` (la primera clase base).
3. `C` (la segunda clase base).
4. `A` (la clase base com煤n).
5. `object` (la clase base de todas las clases en Python).

Puedes ver el MRO usando el m茅todo `mro()` o el atributo `__mro__`:

```python

# Salida: [<class '__main__.D'>, <class '__main__.B'>, <class '__main__.C'>, <class '__main__.A'>, <class 'object'>]
```

En este ejemplo, cuando llamas a `objeto.metodo()`, Python sigue el MRO y ejecuta el m茅todo de la clase `B`, porque `B` aparece antes que `C` en el orden de herencia.
